## What It Does

- Posts random The Thing GIFs to a configured channel at regular intervals
- Admins configure the channel and posting frequency with `/setuptf4`
- Persists guild configurations across bot restarts
- Supports unlimited servers (multi-guild)
- Automatically recovers from errors (deleted channels, missing permissions)

## Setup

### Prerequisites
- Node.js 22.21.1 or later
- A Discord bot token (from [Discord Developer Portal](https://discord.com/developers/applications))
- A Discord server where you can manage permissions

### Installation

1. Clone or download this repo
2. Install dependencies:
   ```bash
   npm install
   ```

3. Create a `.env` file (copy from `.env.example`):
   ```
   DISCORD_TOKEN=your_bot_token_here
   CLIENT_ID=your_application_client_id_here
   ```

4. Register the slash command with Discord:
   ```bash
   npm run deploy
   ```
   This runs once to register `/setuptf4` globally (takes up to 1 hour to propagate to all guilds).

### Running the Bot

```bash
node src/index.js
```

The bot will log in and re-hydrate any saved guild configurations from the last session.

## Usage

In any Discord server where the bot has permissions:

```
/setuptf4 channel:#your-channel interval:120
```

**Options:**
- `channel` — The text channel to post GIFs in
- `interval` — Seconds between posts (must be between 120 and 600)

**Permissions Required:**
- You must have the **Manage Server** permission to run this command
- The bot must have **Send Messages** permission in the target channel

The bot will immediately post one test GIF so you can verify it's working, then continue posting at your configured interval.

## How It Works

### On Startup
1. Loads all saved guild configs from `src/data/guilds.json`
2. Re-hydrates schedulers for each enabled guild
3. Starts posting GIFs at configured intervals

### When `/setuptf4` is Run
1. Verifies the user has **Manage Server** permission
2. Checks the bot can send messages in the target channel
3. Saves the config (channel ID + interval)
4. Starts/restarts the scheduler for that guild
5. Posts one test GIF immediately

### Periodic Posting
- A `setInterval` runs for each guild independently
- On each interval, fetches a random GIF and posts it
- If the channel is deleted or the bot loses access, the scheduler auto-stops

### Error Recovery
The bot gracefully handles:
- Missing or deleted channels → stops the guild's schedule
- Lost permissions → stops the guild's schedule
- Any other errors → logs and keeps the interval running
- Bot restarts → configs are re-loaded and schedules resume

## Security

## Troubleshooting

**Bot doesn't respond to `/setuptf4`**
- Command registration can take up to 1 hour. Wait and refresh Discord.
- If still not showing, re-run `npm run deploy`.

**GIFs aren't posting**
- Check bot has **Send Messages** permission in the target channel
- Check the interval isn't too long and the bot process is still running
- Check the console for error messages

**Schedule keeps stopping**
- If the configured channel was deleted, run `/setuptf4` again with a valid channel
- If the bot lost permissions, update bot permissions and restart the bot
